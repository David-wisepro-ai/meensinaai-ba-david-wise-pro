# 8-Point Quality Criteria — Review Checklist

> Checklist final do Copy Chief Cyrus antes de devolver qualquer copy pro usuario. Toda copy passa por aqui — sem excecao. Falhar 2+ pontos = volta pra revisao.

## Os 8 testes

### 1. Schwartz Test — Headline para o leitor?
- A primeira linha faz o leitor parar de rolar?
- Esta calibrada pro awareness level diagnosticado?
- Headline pra Most Aware nao deve contar historia. Headline pra Unaware nao deve mencionar produto.
- **Falha:** se voce le e nao sente vontade de continuar, headline falhou.

### 2. Halbert Test — Lead engata nas 3 primeiras frases?
- O lead (3 primeiras frases ou primeiro paragrafo) prende ou solta?
- Especifico ou generico?
- Promete algo concreto ou e abstrato?
- **Falha:** se o lead e blah ("Voce ja parou pra pensar..."), falhou.

### 3. Ogilvy Test — Detalhes especificos e concretos?
- Numeros reais, nomes, datas, lugares?
- "300 magazines em 30 minutos" e Ogilvy. "Rapido e facil" e generico.
- Vague claims = perda de credibilidade.
- **Falha:** se voce pode trocar "produto X" por "produto Y" sem reescrever 90%, e generico demais.

### 4. Sugarman Test — Cada frase puxa a proxima?
- Slippery slide: cada paragrafo termina abrindo curiosidade pra o proximo.
- Curiosidade aberta -> resposta no proximo paragrafo -> nova curiosidade.
- **Falha:** se voce pode parar de ler em qualquer paragrafo sem perder nada, falhou.

### 5. Kennedy Test — Oferta clara e irresistivel?
- O que o leitor recebe? Por quanto? Com qual garantia? Por quanto tempo?
- A oferta tem stack (bonus)? Tem urgencia legitima?
- **Falha:** se ao final do copy o leitor pensa "ok, mas o que exatamente eu compro?", falhou.

### 6. Bencivenga Test — Fascinations carregam curiosidade?
- Bullets vendem curiosidade, nao features.
- "Como" + resultado + tensao = fascination.
- "10 features do produto" = lista. "10 segredos que voce so descobre depois de comprar" = fascinations.
- **Falha:** se bullets sao features list, falhou.

### 7. Carlton Test — Fechamento com urgencia + CTA?
- Urgencia legitima (escassez real, deadline real, bonus real)?
- CTA explicito ("clique aqui", "responda este email", "ligue agora")?
- Carlton: "se voce nao escreve uma CTA que parece arma apontada, nao escreveu fechamento".
- **Falha:** CTA passivo ou ausente.

### 8. Universal Test — Voce mesmo compraria?
- Lendo do zero, sem saber quem escreveu, voce compraria?
- Se nao, qual objection nao foi tratada?
- **Falha:** se voce nao compraria, prospect tambem nao compra.

## Como aplicar

1. Leia copy completa de uma vez (nao revise enquanto le pela primeira vez)
2. Reaplique os 8 testes na ordem
3. Marque PASS / FAIL por teste
4. Se 0-1 FAIL: aprovado, devolve pro usuario com nota dos pontos fracos
5. Se 2+ FAIL: devolve pra persona refazer com nota dos pontos especificos
6. Se 4+ FAIL: roteamento errado — reavalie awareness/medium e atribua outra persona

## Output esperado do review

Ao final do review, Cyrus retorna:

```
QUALITY REVIEW — [titulo do copy]

Schwartz Test       [PASS/FAIL] — [comentario]
Halbert Test        [PASS/FAIL] — [comentario]
Ogilvy Test         [PASS/FAIL] — [comentario]
Sugarman Test       [PASS/FAIL] — [comentario]
Kennedy Test        [PASS/FAIL] — [comentario]
Bencivenga Test     [PASS/FAIL] — [comentario]
Carlton Test        [PASS/FAIL] — [comentario]
Universal Test      [PASS/FAIL] — [comentario]

VEREDICTO: [APROVADO / REVISAR / REROTEAR]
ACAO RECOMENDADA: [...]
```

## Quando NAO usar

- Copy interno (briefing, draft pre-final, brainstorm)
- Pecas que nao tem objetivo de venda direta (ex: post de comunidade, thread de update)
- Quando usuario pede "rascunho", nao "copy final"

Em outros 100% dos casos, aplique.
